## DHCP란?

DHCP(Dynamic Host Configuration Protocol)는 컴퓨터나 스마트폰이 네트워크에 접속할 때 자동으로 IP 주소를 할당해주는 프로토콜이다.

ex ) 와이파이에 연결했을 때 직접 IP 주소를 입력하지 않아도 인터넷이 되는 이유가 DHCP 때문이다.

## DHCP 동작 과정

DHCP는 DORA라는 4단계로 동작한다.

| 단계 | **보낸 곳** | **설명** |
| --- | --- | --- |
| Discover | 클라이언트 | 클라이언트 | DHCP 서버를 찾아서 IP 주소 할당 요청 |
| Offer | 서버 | 할당 후보 IP 주소를 제시 |
| Request | 클라이언트 | 후보로 제시된 IP 주소 사용을 요청 혹은 IP 주소의 유효 기간 연장을 요청 |
| Acknowledgment | 서버 | 클라이언트의 요청을 받아들임 |
1. Discover (탐색)
    
    클라이언트가 네트워크에 접속하면 DHCP 서버를 찾기 위해 “IP 주소 좀 주세요”라는 요청을 브로드캐스트로 전송한다.
    
    - 출발지 IP : 0.0.0.0
    - 목적지 IP : 255.255.255.255
    
    즉, 아직 자신의 IP가 없어서 전체 네트워크에 도움 요청을 보내는 것이다.
    
2. Offer (제안)
    
    DHCP 서버가 클라이언트에게 IP 주소, 서브넷 마스크, 게이트웨이, DNS 서버 정보, 임대 시간 등을 전송한다.
    
    ex ) 
    
    - IP 주소 : 192.168.0.10
    - 서브넷 마스크 : 255.255.255.0
    - 기본 게이트웨이 : 192.168.0.1
    - DNS 서버 : 8.8.8.8
    
    쉽게 말하면 : “이 주소 사용해볼래?”하고 제안하는 메시지이다.
    
3. Request (요청)
    
    DHCP 서버로부터 받은 IP 주소, 서브넷 마스크, 게이트웨이, DNS 서버 정보, 임대 시간 등을 받겠다고 확정 요청을 보낸다.
    
    즉 : “좋아요 그 IP 사용할게요” 라고 서버에게 알리는 과정이다.
    
4. Acknowledgment (ACK, 승인)
    
    DHCP 서버가 최종 승인 메시지를 보내면 IP 할당이 완료된다.
    
    이제 클라이언트는 정상적으로 임대 시간 동안 할당 받은 IP를 통해 인터넷을 사용할 수 있다.
    

DHCP의 그 밖의 메시지

| 메시지 이름 | **보낸 곳** | **설명** |
| --- | --- | --- |
| Release | 클라이언트 | IP 주소 해제 통지 |
| Negative Acknowledgment (NAK) | 서버 | 이미 사용 중인 IP 주소를 요청했을 때 발생  |
| Inform | 클라이언트 | IP 주소 이외의 설정 정보 요청 |

## IP 할당 구조

### IP 할당이란?

네트워크에 연결된 다양한 기기가 고유한 IP 주소를 부여하는 과정

## IP 구조 할당 방식

**동적 할당**

자동으로 IP 주소를 설정하는 방식
DHCP를 사용, 기기가 켜지면 DHCP 서버에 주소를 요청하고, 서버는 사용 가능한 IP 주소를 임대 시간과 함께 전송한다.

### DHCP 서버가 제공하는 정보

| **항목** | **설명** |
| --- | --- |
| IP 주소 | 장치의 주소 |
| 서브넷 마스크 | 네트워크 범위 구분 |
| 기본 게이트웨이 | 외부 네트워크로 나가는 문 |
| DNS 서버 | 도메인을 IP로 변환 |
| 임대 시간(Lease Time) | IP 사용 가능 시간 |

**정적 할당**

수동으로 IP 주소를 설정하는 방식
클라이언트가 시스템 관리자에서 고정 IP를 받는다.
