# 어셈블리어

- 어셈블리어란 컴퓨터의 CPU가 이해하는 기계어를 사람이 읽기 쉽도록 문자 형태로 표현한 저급 언어입니다.
- CPU 명령어 형태로 직접적으로 제어할 수 있습니다.

---

## 특징

- CPU가 수행하는 명령어를 사람이 읽기 쉬운 형태로 표현하며, 기계와 1 대 1로 대응됩니다.
- 하드웨어를 세밀하게 제어할 수 있어 운영체제, 임베디드 시스템, 드라이버 개발 등 시스템 프로그래밍 분야에서 많이 사용됩니다.
- CPU의 종류에 따라 사용하는 명령어와 문법이 달라지기 때문에 다른 환경으로의 이식성이 낮다는 특징도 가지고 있다.

---

## 어셈블리어의 기본 인스트럭션

### 1. 데이터 전송 명령어

- MOV
    
    데이터를 다른 장소로 복사하는 명령어입니다.
    
- PUSH
    
    데이터를 스택에 저장하는 명령어입니다.
    
    - POP
        
        스택에 저장된 데이터를 꺼내오는 명령어입니다.
        
- XCHG
    
    두 데이터의 값을 서로 교환하는 명령어입니다.
    
- LEA
    
    메모리의 주소를 가져오는 명령어입니다.
    

### 2. 산술 연산 명령어

- ADD / SUB
    
    숫자를 더하거나 빼는 명령어입니다.
    
- MUL / DIV
    
    곱셈과 나눗셈을 수행하는 명령어입니다.
    
- INC / DEC
    
    값을 1 증가 혹은 감소시키는 명령어입니다.
    

### 3. 논리 연산 명령어

- AND / OR / XOR / NOT
    
    비트 단위 논리 연산을 수행하는 명령어입니다.
    
- CMP / TEST
    
    값이나 비트를 비교, 검사하는 명령어입니다.
    

### 4. 흐름 제어 명령어

- 흐름 제어 명령어는 프로그램의 실행 순서를 바꾸는 명령어입니다.
조건에 따라 다른 위치로 이동하거나, 반복문과 함수 호출 등을 처리할 때 사용된다.
    - JMP
        
        지정한 위치로 무조건 이동하는 명령어입니다.
        
    - JE
        
        두 값이 같을 때 이동하는 명령어입니다.
        
    - JNE
        
        두 값이 다를 때 이동하는 명령어입니다.
        
    - JG
        
        왼쪽 값이 더 클 때 이동하는 명령어입니다.
        
    - JL
        
        왼쪽 값이 더 작을 때 이동하는 명령어입니다.
        
    - CALL
        
        함수를 호출하는 명령어입니다.
        
    - RET
        
        함수를 끝내고 원래 위치로 돌아가는 명령어입니다.
        
    - LOOP
        
        반복문을 수행하는 명령어입니다.
        

### 5. 기타 다른 어셈블리어 명령어

- NOP
    
    아무 동작도 하지 않는 명령어입니다.
    
- INT
    
    인터럽트를 호출하는 명령어입니다.
    
- HLT
    
    CPU 동작을 멈추는 명령어입니다.
    
- CLC / STC
    
    Carry Flag 값을 조작하는 명령어입니다.
    
    CLC : Carry Flag를 0으로 설정
    
    STC : Carry Flag를 1로 설정
    
- CLI / STI
    
    인터럽트 허용 여부를 설정하는 명령어입니다.
    
    CLI : 인터럽트 금지
    
    STI : 인터럽트 허용
    
- PUSHF / POPF
    
    플래그 레지스터를 저장하거나 복원하는 명령어입니다.
    
    PUSHF : 플래그 저장
    
    POPF : 플래그 복원
    
- LEAVE
    
    함수 종료를 준비하는 명령어입니다.
