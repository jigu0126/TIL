# 네트워크 보안이 필요한 이유

네트워크를 통해 개인정보, 금융 정보, 기업의 중요 데이터 등이 전송되기 때문에 이를 안전하게 보호하기 위해 네트워크 보안이 필요하다.

네트워크 보안이 없으면 해커가 데이터를 훔치거나 내용을 변경하고, 서버를 마비시키거나 악성코드를 유포할 수 있다. 따라서 안전한 통신 환경을 만들기 위해 다양한 보안 기술이 사용된다.

## 필요한 이유

- 개인정보 및 중요 데이터 보호
- 해킹 및 악성코드 감염 방지
- 서비스 장애 예방
- 안전한 인터넷 사용

---

# 스니핑(Sniffing)

스니핑은 **네트워크를 통해 전송되는 패킷(Packet)을 몰래 수집하여 내용을 확인하는 공격**이다.

암호화되지 않은 데이터를 전송하는 경우 아이디, 비밀번호, 개인정보 등을 쉽게 탈취할 수 있다.

## 특징

- 네트워크 패킷을 몰래 수집
- 암호화되지 않은 통신에서 위험
- 로그인 정보 및 개인정보 탈취 가능

## 방어 방법

- HTTPS 사용
- SSL/TLS를 이용한 암호화 통신
- VPN 사용

---

# ARP Spoofing

ARP Spoofing은 **가짜 ARP 응답을 보내 자신의 MAC 주소를 다른 장치의 MAC 주소인 것처럼 속이는 공격**이다.

ARP는 **IP 주소와 MAC 주소를 연결해 주는 프로토콜**이다. 공격자는 자신의 MAC 주소를 공유기(게이트웨이)의 MAC 주소처럼 속여 피해자의 데이터를 자신에게 먼저 전달하도록 만든다.

## 동작 과정

1. 피해자가 공유기의 MAC 주소를 찾는다.
2. 공격자가 "내가 공유기다."라는 가짜 ARP 응답을 보낸다.
3. 피해자의 ARP 테이블이 변경된다.
4. 피해자의 모든 데이터가 공격자를 거쳐 공유기로 전달된다.

## 특징

- ARP 테이블을 변조
- 패킷을 공격자가 먼저 받음
- 스니핑 및 MITM 공격에 자주 사용

## 방어 방법

- Static ARP 사용
- Dynamic ARP Inspection(DAI)
- ARP 감시 프로그램 사용

# DoS(Denial of Service)

DoS 공격은 **한 대의 컴퓨터가 서버에 과도한 요청을 보내 서비스를 사용할 수 없게 만드는 공격**이다.

서버가 처리할 수 있는 양보다 훨씬 많은 요청을 보내 정상적인 사용자가 서비스를 이용하지 못하도록 만든다.

## 특징

- 공격자 1대
- 서버 과부하 발생
- 서비스 이용 불가

## 방어 방법

- 방화벽(Firewall) 사용
- 요청 속도 제한(Rate Limiting)
- IDS/IPS를 이용한 공격 탐지 및 차단

---

# DDoS(Distributed Denial of Service)

DDoS 공격은 **여러 대의 컴퓨터(좀비 PC)가 동시에 서버를 공격하는 방식**이다.

공격자는 악성코드에 감염된 수많은 컴퓨터를 이용하여 동시에 공격하기 때문에 DoS보다 규모가 크고 방어하기 어렵다.

## 특징

- 여러 대의 컴퓨터가 동시에 공격
- 대규모 트래픽 발생
- 탐지 및 방어가 어려움

## DoS와 DDoS 비교

| DoS | DDoS |
|------|------|
| 공격자 1대 | 공격자 여러 대(봇넷) |
| 공격 규모가 작음 | 공격 규모가 매우 큼 |
| 방어가 비교적 쉬움 | 방어가 어려움 |

## 방어 방법

- DDoS 전용 방어 장비 사용
- CDN 및 클라우드 기반 DDoS 방어 서비스
- 트래픽 필터링 및 로드 밸런싱

---

# MITM(Man In The Middle)

MITM(중간자 공격)은 **통신하는 두 사용자 사이에 공격자가 끼어들어 데이터를 가로채거나 변경하는 공격**이다.

사용자는 정상적으로 서버와 통신한다고 생각하지만 실제로는 공격자를 거쳐 데이터를 주고받게 된다.

## 특징

- 통신 내용 도청
- 데이터 변조 가능
- 사용자와 서버 모두 공격 사실을 알아채기 어려움

## 대표적인 공격 방법

- ARP Spoofing
- DNS Spoofing
- 가짜 Wi-Fi(AP)

## 방어 방법

- HTTPS 사용
- SSL/TLS 사용
- VPN 사용
- 인증서 확인

---

# DNS Spoofing

DNS Spoofing은 **DNS 응답을 위조하여 사용자를 가짜 웹사이트로 접속시키는 공격**이다.

사용자는 정상적인 주소를 입력했지만 공격자가 조작한 DNS 정보를 받아 피싱 사이트나 악성 사이트로 이동하게 된다.

## 특징

- DNS 응답 위조
- 가짜 홈페이지로 접속 유도
- 개인정보 및 계정 정보 탈취

## 방어 방법

- DNSSEC 사용
- HTTPS 확인
- 신뢰할 수 있는 DNS 서버 사용
- VPN 사용
