# 레지스터란?

---

디지털 회로에서, 레지스터랑 플립플롭이 여러 개로 구성되어 여러 비트의 정보를 저장하는 회로이다.
컴퓨터 구조에서, 프로세서에 위치한 고속 메모리로 극히 소량의 데이터나 처리 중인 중간 결과와도
같은 프로세서가 바로 사용할 수 있는 데이터를 담고 있는 영역을 레지스터라고 한다.

## 레지스터의 종류와 역할

## 1. 범용 레지스터

---

- 법용 레지스터는 연산과 결과의 임시 저장, 산술 및 논리 연산, 주소 색인 등 다양한 용도로 사용되는 다목적 레지스터이다.
    - EAX (Accumulator Register): 산술, 논리 연산을 담당하는 레지스터로, 함수의 반환값이 이 레지스터에 저장된다.
    - EBX (Base Register): 메모리 주소를 저장하기 위해 사용되는 레지스터, 종종 배열이나 문자열과 같은 데이터 구조에 접근하기 위한 기준 포인터로 사용된다.
    - ECX (Count Register): 반복 작업에서 카운터 역할을 수행하는 레지스터이다.
    - EDX (Data Register): EAX 레지스터와 함께 사용하여 큰 수를 연산을 하거나, 그 결과를 저장할 수 있는 레지스터이다.

### 인덱스 레지스터

- 메모리 내의 데이터 접근 및 조작에 특화된 레지스터이다.
    - ESI (Source Index): 데이터 복사, 문자열 연산, 입력/출력 처리 등의 작업에서 
    소스 데이터의 주소를 가리키는데 사용된다.
    - EDI (Destination Index): 데이터 복사, 문자열 처리, 배열 조작 등의 작업에서 목적지 데이터의 메모리 주소를 가리키는데 사용된다.

### 포인터 레지스터

스택과 프로그램의 실행 흐름 관리에 사용되는 레지스터로, 메모리 주소가 저장된다.

- ESP (Stack Pointer): 프로그램의 스택 메모리 내에서, 
현재 스택 최상단 주소를 저장하는 레지스터이다.
- EBP (Base Pointer): 함수 내의 지역 변수와 인자에 일관되고, 
쉽게 접근하기 위해 사용되는 포인터 역할 레지스터이다.
- EIP (Instruction Pointer): 다음에 실행될 명령의 메모리 주소를 저장한다.

## 2. 제어 레지스터

### 프로그램 실행 방식, 메모리 접근, 보호 기능 등을 ‘통제’하는 레지스터이다.

## 3. 세그먼트 레지스터

---

---

- 메모리를 다른 세그먼트로 나누어 관리하고, 각 세그먼트에 대한 기준 주소를 저장한다.
세그먼트를 이용함으로써, 물리 메모리를 효율적으로 사용하고 프로그램 간 메모리 격리 가능하게 한다.
    - CS (Code Segment): 현재 프로그램의 코드가 포함된 세그먼트의 주소를 저장하는 레지스터
    - DS (Data Segment): 데이터가 포함된 세그먼트의 주소를 저장하는 레지스터
    - SS (Stack Segment): 스택이 포함된 세그먼트의 주소를 저장하는 레지스터
    - ES, FS, GS: 추가적인 데이터 <-> 세그먼트 주소를 저장하는 레지스터

## 4. 플래그 레지스터

- 연산의 결과를 나타내는 플래그들을 포함하고, 특정 프로세서 연산을 제어한다.
    - ZF (Zero Flag): 작업 결과가 0일 경우 참
    - SF (Sign Flag): 연산 결과가 음수일 경우 참
    - CF (Carry Flag): 가장 높은 자리에서 자리올림이 발생할 경우 혹은 계산 중 빌림이 발생할 경우 참
    - OF (Overflow Flag): 표현 범위를 넘어섰을 경우 참
    - PF (Parity Flag): 결과에 설정된 비트 수가 짝수일 겨우 참
    - IF (Interrupt Enable/Disable Flag): 인터럽트를 활성화 할지 말지 결정
