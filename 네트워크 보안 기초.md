# 네트워크 보안 기초

## 네트워크 보안 기초

네트워크 보안은 외부 공격이나 허가되지 않은 접근으로부터 네트워크와 데이터를 보호하는 기술이다.

정보의 **기밀성(Confidentiality)**, **무결성(Integrity)**, **가용성(Availability)** 을 유지하는 것이 가장 중요한 목적이다.

### 네트워크 보안의 목적

- 허가되지 않은 접근 차단
- 데이터 유출 방지
- 데이터 위조 및 변조 방지
- 서비스 중단 방지
- 안전한 통신 제공

---

## 방화벽 (Firewall)

방화벽은 내부 네트워크와 외부 네트워크 사이에서 들어오고 나가는 패킷을 검사하여 허용하거나 차단하는 보안 장비이다.

미리 설정된 보안 규칙(정책)에 따라 통신을 허용하거나 차단하여 내부 네트워크를 보호한다.

### 특징

- 내부 네트워크 보호
- 허가되지 않은 접근 차단
- 패킷 검사 후 허용 또는 차단
- 보안 정책 적용

### 종류

- **패킷 필터링(Packet Filtering)** : IP 주소, 포트 번호 등을 확인하여 허용 또는 차단
- **Stateful Inspection** : 연결 상태까지 확인하여 검사
- **Proxy Firewall** : 사용자 대신 통신하여 내부 정보를 보호
- **차세대 방화벽(NGFW)** : 애플리케이션까지 분석하여 보안 강화

---

## ACL (Access Control List)

ACL은 네트워크 장비에서 특정 사용자의 접근을 허용하거나 차단하는 규칙이다.

IP 주소, 포트 번호, 프로토콜 등을 기준으로 접근 권한을 제어한다.

### 특징

- 접근 허용 및 차단
- 라우터와 스위치에서 많이 사용
- 방화벽에서도 사용
- 규칙의 순서가 매우 중요
- 위에서 아래 순서대로 검사

### 예시

- 192.168.10.0/24 대역의 HTTP 통신 허용
- 외부에서 SSH(22번 포트) 접속 차단

---

## IDS (Intrusion Detection System)

IDS는 네트워크를 감시하여 공격이나 비정상적인 활동을 탐지하는 보안 시스템이다.

공격을 발견하면 관리자에게 알려주지만 직접 차단하지는 않는다.

### 특징

- 공격 탐지
- 관리자에게 경고
- 패킷 분석
- 차단 기능 없음

### 장점

- 공격을 빠르게 발견 가능
- 네트워크 상태를 지속적으로 감시

---

## IPS (Intrusion Prevention System)

IPS는 IDS와 비슷하지만 공격을 탐지한 후 자동으로 차단까지 수행하는 보안 시스템이다.

실시간으로 패킷을 검사하여 공격을 막는다.

### 특징

- 공격 탐지
- 자동 차단
- 악성 트래픽 차단
- 실시간 대응

### IDS와 IPS 차이

| IDS | IPS |
| --- | --- |
| 공격을 탐지만 함 | 공격을 탐지하고 차단 |
| 관리자에게 경고 | 자동으로 공격 차단 |
| 네트워크 감시 | 네트워크 보호 |

---

## VPN (Virtual Private Network)

VPN은 인터넷을 통해 안전한 사설 네트워크를 만드는 기술이다.

데이터를 암호화하여 외부에서 내용을 볼 수 없도록 보호한다.

### 특징

- 데이터 암호화
- 원격 접속 가능
- 안전한 통신
- 공용 Wi-Fi에서도 안전하게 사용 가능

### 사용하는 곳

- 재택근무
- 회사 내부망 접속
- 해외에서 국내 서비스 이용
- 공공장소 인터넷 사용

---

## HTTPS (HyperText Transfer Protocol Secure)

HTTPS는 HTTP에 SSL/TLS 암호화를 적용한 안전한 웹 통신 방식이다.

웹 브라우저와 웹 서버 사이의 데이터를 암호화하여 도청이나 변조를 막는다.

### 특징

- 데이터 암호화
- 서버 인증
- 데이터 무결성 보장
- URL 앞에 **https://** 사용
- 기본 포트 번호는 **443번**

### HTTP와 HTTPS 차이

| HTTP | HTTPS |
| --- | --- |
| 암호화 없음 | SSL/TLS로 암호화 |
| 보안이 약함 | 보안이 강함 |
| 포트 80 | 포트 443 |

---

## SSL / TLS

SSL과 TLS는 인터넷에서 데이터를 암호화하기 위한 보안 프로토콜이다.

현재는 SSL의 보안 취약점을 개선한 TLS를 사용하며, HTTPS도 TLS를 이용하여 데이터를 암호화한다.

### 특징

- 데이터 암호화
- 서버 인증
- 데이터 무결성 확인
- 안전한 인터넷 통신 제공

### SSL과 TLS 차이

| SSL | TLS |
| --- | --- |
| 오래된 보안 프로토콜 | SSL을 개선한 최신 보안 프로토콜 |
| 현재 거의 사용하지 않음 | 현재 대부분 사용 |
| 보안 취약점 존재 | 보안 강화 |
