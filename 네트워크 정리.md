# 네트워킹이란?

---

- 네트워킹이란(Networking) : 장비들이 서로 대화할 수 있게 연결해주는 것
- 네트워킹의 목적 : 장비, 정보의 공유

# 인터넷, 인트라넷, 엑스트라넷

## 인터넷

---

- 인터넷(Internet) : 여러 개의 네트워크를 묶은 것

### 인터넷의 특징

---

1. 하나의 프로토콜을 사용 (TCP / IP) 프로토콜이 다르면 통신이 불가하기 때문
2. 웹 브라우저(구글 크롬, 마이크로소프트 엣지, 등)을 사용
3. 인터넷에는 없는 정보 X

## 인트라넷

---

- 인트라넷(Intranet) : 내부의 네트워크
    - TCP / IP 프로토콜 사용
    - 웹 브라우저를 이용하여 사내 업무 처리
    - 회사 직원만 사용 가능

## 엑스트라넷

---

- 엑스트라넷(Extranet) : 기업의 종업원 이외에 협력사와 고객이 이용할 수 있는 네트워크

# LAN(Local Area Network)이란?

---

- LAN : 어느 한정된 곳에서 네트워크를 연결
- WAN(Wide Area Network) : 멀리 떨어진 곳에서 네트워크를 연결

# 이더넷

---

- 이더넷(Ethernet) : 네트워킹의 한 방식
- CSMA(Carrier Sense Multiple Access) / CD(Collision Detection) 프로토콜 사용
- CSMA / CD : 대충 알아서 눈치로 통신
    - 이더넷 환경에서 통신을 하고 싶은 PC나 서버는 먼저 지금 네트워크상에 통신이 일어나고 있는지 확인한다. → Carrier Sense(Carrier : 네트워크 상에서 나타나는 신호)
    - Carrier가 감지되면 대기한다 그러다가 통신(Carrier)이 사라지면 통신한다.
    - 네트워크 상에서 통신이 일어나지 않을 때 두 PC나 서버 자신의 데이터를 동시에 보낼 수 있다.
        
        → Multiple Access(다중 접근)
        
    - Multiple Access(다중 접근)이 발생하면 충돌(Collision)이 발생한다. 이런 경우가 일어나지 않게 점검하는 것 → Collision Detection(충돌 감지)
    - 충돌이 발생하면 랜덤한 시간 이후 다시 데이터를 전송한다. (15회 반복)

# 토큰링이란?

---

- 토큰링이란(TokenRing) : 오직 한 PC, 즉 토큰을 가진 PC만 통신하는 네트워크 방식
- 통신이 끝나면 옆 PC에 토큰 전달
- 장점 → 충돌(Collision)이 일어나지 않는다.
- 단점 → 차례를 기다려야 한다.

# UTP 케이블

---

- UTP(Unshielded twisted-pair)
    - 우리가 주로 사용하는 케이블, 언쉴드는 감싸지 않았다는 뜻
- STP(Shielded twisted-pair)
    - 케이블 주위를 절연체로 감싸서 보호
    - UTP보다 비싸고 성능 좋음
    - 토큰링쪽에서 많이 쓰임
- 카테고리 1 : 주로 전화망에 사용, 따라서 데이터 전송용으로 부적합
- 카테고리 2 : 데이터를 최대 4Mbps의 속도로 전송하는 케이블
- 카테고리 3 : 10 Base T 네트워크에 사용되는 케이블, 최대 10Mbps
- 카테고리 4 : 토큰링 네트워크에서 사용되는 케이블, 최대 16Mbps
- 카테고리 5 : 최대 100Mbps, 기가비트도 가능
- 카테고리 6 : 기가비트 이상의 속도에 적합한 케이블, 최근 사용하는 케이블 중 가장 많은 종류
- 카테고리 7 : 10Gbps에 적합한 케이블

# 케이블

---

- 10 Base T
- 10 : 100Mbps의 속도를 지원하는 케이블
- Base : Baseband용 케이블(Baseband, 디지털 방식 / 아날로그 방식, Broadband)
- T : TP(Twisted Pair) 케이블 (케이블의 종류 또는 케이블이 전송할 수 있는 최대 거리를 보여주는 자리)

# 맥 어드레스

---

- MAC(Media Access Control) : 네트워크 상에서 서로를 구분하는 주소
- IP 주소를 MAC으로 바꾸는 절차(ARP, Address Resolution Protocol)
- 48bit, 6옥텟(octet) → 12자리의 16진수 맥 어드레스
    - 옥텟(octet)은 8개의 비트의 모임
- 앞에서 6개의 16진수가 벤터, 생산자를 나타내는 코드로 “OUI(Organizational Unique Identifier)”라고 한다.

# 유니캐스트, 브로드캐스트, 멀티캐스트

---

- 유니캐스트(Unicast)
    - 현재 네트워크 상에서 가장 많이 사용되는 통신 방식
    - 특정 주소 하나만을 가지고 통신하는 통신 방식
    - 다른 PC의 CPU에 영향을 주지 않는다.
        - 자신의 랜카드 맥 어드레스가 아니면 바로 이 프레임을 버린다.
        - 자신의 랜카드 맥 어드레스와 같으면 이 프레임을 CPU로 전달한다.
- 브로드캐스트(Broadcast)
    - 로컬 랜에 붙어 있는 모든 네트워크 장비들에게 보내는 통신 방식
        - 로컬 랜 : 라우터에 의해 분리된 공간, 브로드캐스트 도메인(영역)
    - 브로드캐스트의 주소는 FFFF.FFFF.FFFF(맥 어드레스)로 정해져 있음.
        - 이 주소를 받은 랜카드는 자신의 맥 어드레스와 같지 않아도 패킷을 CPU에 전달함
        - 랜카드의 맥 어드레스가 다른 PC의 CPU 성능도 떨어뜨림
    - IP를 MAC Address로 바꾸는 과정 : ARP(Address Resolution Protocol) → 브로드캐스ㅌ
        1. IP 주소를 가진 PC를 찾기 위해 브로드캐스트를 보냄
        2. 그 주소를 가진 PC가 자신의 맥 어드레스를 보냄
- 멀티캐스트(Multicast)
    - 보내고자 하는 그룹 멤버들에게만 한 번에 보낼 수 있는 통신 방식
    - 유니캐스트와 브로드캐스트의 장점을 결합한 방식
    - 스위치나 라우터가 이 멀티캐스트 기능을 꼭 지원해야 한다는 제약이 있음

# OSI 7 Layer

---

- 통신의 관한 국제적인 표준기구인 International Organization for Standardization(ISO)에서 만듬.
- 통신을 7개의 단계별로 표준화하여 그 효율성을 높이기 위해 사용
    - Application Layer(애플리케이션 개층)
    - Presentation Layer(프레젠테이션 계층)
    - Session Layer(세션 계층)
    - Transport Layer(트랜스포트 계층)
    - Network Layer(네트워크 계층)
    - Data Link Layer(데이터 링크 계층)
    - Physical Layer(피지컬 계층)
- 왜?
    1. 데이터의 흐름이 한눈에 보임
    2. 문제가 발생하면 7개의 작은 문제를 나눠 문제 해결이 편리함
    3. 여러 회사 장비를 써도 네트워크가 이상 없이 돌아감
- 피지컬 계층(Physial Layer)
    - 통신의 맨 아래 단계
    - 전기적, 기게적, 기능적인 특성을 이용해 통신 케이블로 데이터를 전송
    - 통신 단위는 비트(bit)
    - 대표적인 장비 → 통신 케이블, 리피터, 허브 등
- 데이터 링크 계층(Data Link Layer)
    - 피지컬 레이어를 통해 송·수신되는 정보의 오류와 흐름을 관리 → 안전한 정보의 전달을 도움
    - 맥 어드레스를 가지고 통신할 수 있게 해줌
    - 이 계층에서 전송되는 단위 → 프레임
    - 대표적인 장비 → 브리지, 스위치 등
- 네트워크 계층(Network Layer)
    - 데이터를 목적지까지 가장 안전하고 빠르게 전달 → 라우팅
    - 경로를 선택, 주소 지정, 경로에 따라 패킷을 전달해주는 역할
    - 장비 → 라우터, 라우팅 기능을 수행하는 스위치(Layer 3 스위치)
- 트랜스포트 계층(Transport Layer)
    - 플로 컨트롤과 에러 복구 기능 → 에러 복구를 위해 패킷 재전송, 플로를 조절해서 데이터를 정상적으로 전달시키는 역할
    - TCP나 UDT가 이 계층에 해당
- 헤더 : 각 계층별로 관리하는 여러 가지 관리 사항들이 붙음
    - 헤더와 같은 정보가 붙으면서 데이터가 처음 만들어진 데이터보다 커짐

# 프로토콜

---

- 프로토콜(Protocol) : 우리말로 규약, 협약 → 프로토콜이 같은 것끼리만 통신 가능
- TCP(Transmission Control Protocol) / IP(Internet Protocol) : 인터넷에서 사용하는 프로토콜
- IPX(Internetwork Packet eXchange) : 같은 게임방에서 친구들끼리 편을 나누어 게임을 할 때 사용 → 게임을 할 수 있는 건 이 프로토콜을 사용하기 때문
- Appletalk : “맥”이라고 부르는 매킨토시들이 서로 간의 통신을 위해 사용하는 프로토콜

# TCP / IP

---

- TCP(Transmission Control Protocol) / IP(Internet Protocol) → 인터넷 사용을 위해 꼭 필요한 프로토콜

# 나만의 고유한 것

---

- IP 주소  : 인터넷을 사용하는 모두에게 하나씩 주어진 주소
    - 전 세게적으로 유일한 주소를 가지게 됨
    - 똑같은 주소를 갖게 되면 둘 중 한 명은 인터넷 접속 불가능
- 인터넷을 사용하는 사람들이 점점 늘어나 사용할 수 있는 주소가 부족
    - NAT(Network Address Translation) : 내부 네트워크에선 공인되지 않은 IP 주소, 인터넷으로 나갈 때만 공인 주소(유일한 IP 주소)를 가지고 나가는 방식
    - PAT : 동일한 IP 주소를 가지고 여러 명이 인터넷에 접속하면서 포트를 넘버만 바꾸는 방식
    - NIC(Network Informaion Center) : 유일한 IP주소를 사용하기 위해 공인된 IP 주소를 관리하고 나눠주는 기관
- 네 자리의 십진수로 이루어졌고 중간에 점을 하나씩 찍어주는 형태(10.139.4.36)
- 1부터 255까지 가능 → 여기서의 십진수 한 자리인 이진수 8자리가 만들어낼 수 있는 가장 큰 수가 255(11111111)

# 컴퓨터는 이진수로만 이해한다

---

- IP 주소는 원래 이진수로 되었지만, 사람들이 이해하기 쉬우라고 십진수를 만들어 쓴다.
- IP 주소 방식은 기존 버전4(IPv4)와 주소가 부족해 만들어진 버전6(IPv6)이 있다.

# 이진수 계산

---

- Logical AND : 양쪽이 모두 1인 경우에만 1이 되고, 하나라도 0인 경우에는 0이 되게 하는 기능
- TCP / IP 주소 규칙을 이해하려면 이진수에 대한 완벽한 이해 필요

- DHCP(Dynamic Host Configuration Protocol) : IP주소를 자동으로 배정
    1. DHCP 서버가 그 네트워크에 필요한 IP 주소를 전부 가지고 있음
    2. IP 주소를 요구하는 PC에는 그때그때 자동으로 분배
    3. 다 쓴 IP 주소는 회수

# 랜카드

---

- NIC(Network Interface Card) : 랜카드, 주변에서 가장 많이 볼 수 있는 네트워크 장비
    - 유저의 데이터를 케이블에 실어서 허브나 스위치, 라우터 등으로 전달해주고 자신에게 온 데이터를 CPU에게 전달해주는 역할
    - 플러그 앤 플레이(Plug and Play 또는 P&P) : PC에 랜카드를 꽂고 전원을 껐다가 키면 PC가 알아서 랜카드를 인식하고, 알아서 필요한 프로그램을 설치하고 네트워크에 연결해주는 기능
    - 자신이 가지고 있는 PC에 맞는 걸 구입
    ex) 데스크톱, 노트북
    - PC의 버스 방식에 맞는 것을 설치
    ex) PCI 방식, ISA(아이사), EISA(Enhanced ISA
        - 버스(Bus) : 컴퓨터에서 데이터가 날아다니는 길
- 속도에 따른 길 구분
    - 10메가, 10/100메가, 1기가
    - 요즘은 10Mbps 또는 1Gbps용 랜카드가 대부분
    - 이론적인 속도보다 실제 속도는 더 느림 → 데이터가 이동되며 여러 헤더 파일이 붙기 때문
- 케이블 종류에 따른 구분
    - TP 포트를 가진 랜카드, 가장 일반적으로 사용
    - BNC나 AUI 포트를 가진 랜카드
    - 광케이블과 접속하는 랜카드
        - 콤보(Combo) 방식 : AUI와 BNC, 그리고 UTP를 모두 골라 연결할 수 있는 타입
- IRQ(인터럽트 요청), Base Memory(메모리 범위)

# 허브

---

- 허브(Hub) → 멀티포트(Multiport) 리피터(Repeater), 포트가 여러 개 달려 한 포트로 들어온 데이터를 나머지 모든 포트로 뿌려주는 장비, 연결된 모든 PC들이 서로 통신할 수 있게 해줌
    - 속도에 따라 10Mbbps 허브와 100Mbps(패스트 이더넷) 허브가 있음
- 리피터 : 네트워크에서 데이터를 전송하는 경우 케이블에 따라 전송 거리에 제약이 있는데 두 장비간 거리가 제약의 거리보다 멀 때, 중간에 들어온 데이터를 다른 쪽으로 전달해주는 역할을 하는 것
    - 현재는 허브가 리피터의 기능을 가지고 있어 허브가 리피터를 대체
- 이더넷 허브는 CSMA / CD의 적용을 받음
    - 콜리전 도메인(Collision Domain) : 같은 허브에 연결되어 있는 모든 PC들이 있는 영역
- 인텔리전트(Intelligent) 허브 : 지능형 허브, NMS(네트워크 관리 시스템)에서 모든 데이터를 분석, 제어함, 멀리 있는 허브의 동작을 감시하고 조정 가능
    - 대형 네트워크에서 NMS를 쓰는 경우에나 필요함
    - 어떤 PC가 계속 이상한 데이터를 허브로 끊임없이 보내 다른 PC들까지 통신이 불가한 경우, 문제의 PC가 연결된 포트를 찾아내어 자동으로 Isolation(현 네트워크에서 분리시켜서 따로 고립 시킴)시켜 버림)
- 세미(Semi) 더미 허브 : 인텔리전트 허브가 될 수 있는 더미 허브

## 허브의 한계

---

- 아무리 빠른 속도를 내는 허브를 쓴다고 해도 어느 한 순간에는 한 PC만 데이터를 보낼 수 있음
- 허브에 연결된 PC가 많아질수록 콜리전 도메인이 커져 콜리전이 더 자주 발생함

## 허브의 끝과 스위치의 시작

---

- 스위치 : 포트별로 도메인을 나누어 여러 PC가 동시에 통신할 수 있게 해주는 장비
    - 스위치의 경우 각각의 포트에 연결되 ㄴPC가 독자적으로 10Mbps, 100Mbps 속도를 갖는다 함.
- 스위치와 허브의 장점
    - 스위치는 허브가 처리할 수 없는 콜리전 문제의 해결, 에러 처리 능력이 우수
    - 허브는 스위치보다 싸고, 데이터 처리 속도가 일반적으로 스위치에 비해 빠름
- 상황에 따라 스위치와 허브를 골라 쓰는 것이 중요
    - 스위치의 가격이 많이 싸지고 모두 인터넷을 이용하여 많은 데이터를 주고받는 추세라서 스위치가 각광을 받고 있음

# 브리지

---

- 브리지 : 스위치의 원조 격, 허브로 만들어진 콜리전 도메인 사이를 반으로 나누고 중간에 다리를 놓음으로써 도메인을 나눔
    
    → 다리 남단은 남단끼리, 북단은 북단끼리 동시에 통신 가능
    
- 남단에 있는 PC와 북단에 있는 PC가 서로 통신하고자 할 때만 다리를 건너서 통신이 이루어짐

## 브리지 / 스위치의 기능

---

- Learning : 출발지의 맥 어드레스(MAC Address)를 브리지 테이블에 저장함
- Flooding : 프레임의 맥 어드레스가 브리지 테이블에 없는 경우, 그 주소를 나머지 포트로 뿌림
- Forwarding(포워딩) : 목적지의 맥 어드레스가 브리지 테이블에 있고, 출발지와 다른 세그먼트에 있을 때, 해당 포트 쪽으로만 프레임을 뿌려주는 것
    
    → 콜리전 도메인을 나눠주는 기능
    
- Filtering(필터링) : 브리지가 목적지의 맥 어드레스를 알고, 출발지와 목적지가 같은 세그먼트에 있을 때, 브리지를 건너지 않아도 통신이 일어날 수 있으므로 브리지를 막는 것
- Aging : 맥 어드레스를 브리지 테이블에 저장하고 300초가 지나도록 그 출발지 주소를 가진 프레임이 들어오지 않으면 브리지 테이블에서 삭제시키는 것(시간은 조정 가능)

# Looping

---

- Looping : 프레임이 네트워크 상에서 돌아서 데이터 전송이 불가해지는 상태 (이더넷 특성 때문)
- 브리지나 스위치에 목적지까지의 경로가 두개 이상 존재하면 반드시 루핑 발생
- 스패닝 트리 알고리즘(Spanning Tree Algorithm) : 자동으로 루핑을 막아주는 알고리즘

# 스패닝

---

- 스패닝 트리 알고리즘(Spanning Tree Algorithm) : 스위치나 브리지에서 루핑이 발생하는 것을 미리 막기 위해 두 개 이상의 경로가 발생하면 하나의 경로를 제외하고 나머지 경로를 자동으로 막음
    - 기존 경로에 문제가 생기면 막아놓은 경로를 풀어서 데이터를 전송
- 이더 채널(Ether-Channel) 기술 : 여러 개의 링크를 마치 하나의 링크처럼 인식되게 하는 기술
    
    → 하나의 링크가 끊어져도 기다리는 시간 없이 링크가 유지됨
    
    - 패스트 이더 채널(Fast ether Channel, 100메가로 연결된 포트 묶음)
    - 기가 이더 채널(Giga Ether Channel, 기가 비트 이더넷, 즉 1,000메가로 연결된 포트 묶음)
- 업 링크 패스트(Uplink Fast) : 전통적인 스패닝 트리에서 링크의 복구 시간이 1분 이상 걸리므로 복구 시간을 약 2~3초 안에 가능하도록 함

# 라우팅? or 스위칭?

---

- 라우터 : 브로드캐스트 영역(도메인)을 반으로 나눠주는 역할을 함
- 노드 : 네트워크에 접속되는 장비, 주로 PC의 숫자가 됨
    - 브로드캐스트 영역을 나눌 때,
    IP일 때 약 500 노드
    IPX일 때 300 노드
    AppleTalk는 200 노드 정도가 좋다
    그러나 실질적으로 저 기준에서 약 반 정도만 사용해야 제대로 네트워크를 사용할 수 있음
- 스위치나 라우터를 따로 쓰는 것이 아닌 스위치와 라우터를 적당히 사용해야 함

# IP 주소

---

- IP 주소는 원래 이진수 32자리이고, 각 8자리(십진수로 최대 255), 즉 옥텟(octet) 사이에 점을 찍음
- 우리가 보통 쓰는 IP 주소는 이진수를 다시 십진수로 만들어서 쓰는 방식

---

- 모든 IP 주소는 네트워크 부분과 호스트 부분으로 나뉨
- “한 네트워크” : 하나의 브로드캐스트 영역, 라우터를 거치지 않고 통신이 가능한 영역
- “한 네트워크상”에서 뒤에 오는 호스트 부분은 각 PC가 모두 달라야 하고 앞에 오는 네트워크 부분은 같아야 한다
- IP 주소에 네트워크 부분만 라우터가 라우팅을 할 때 참고함

---

- 클래스 A
    - 하나의 네트워크가 가질 수 있는 호스트 수가 가장 많은 클래스
    - 32개의 이진수 중에 맨 앞쪽 하나가 항상 0으로 시작되는 것들
    - ex) 0XXX XXXX.XXXX XXXX.XXXX XXXX.XXXX XXXX
    - ex) 0000 0000.0000 0000.0000 0000.0000 0000 / 0.0.0.0
    - ex ) 0111 1111.1111 1111.1111 1111.1111 1111 / 127.255.255.255
    - 앞의 8비트(1개의 옥텟, 맨 앞의 십진수 부분)가 네트워크 부분, 나머지 24비트(3개의 옥텟, 나머지 3개의 십진수)가 호스트 부분을 나타냄
    - 1 ~ 126으로 시작하는 네트워크
    - X.0.0.0은 X 네트워크 전체를 나타내기 때문에 사용하지 않고, X.255.255.255는 X 네트워크 전체에 있는 모든 호스트들에게 전송할 때 사용하는 브로드캐스트 주소이기 때문에 호스트 주소로 사용하지 않음. 즉 한 네트워크에서 가능한 호스트 수 = 2^24 - 2 = 16,777,214개
- 클래스 B
    - ex) 10XX XXXX.XXXX XXXX.XXXX XXXX.XXXX XXXX
    - ex) 1000 0000.0000 0000.0000 0000.0000 0000 / 128.0.0.0
    - ex) 1011 1111.1111 1111.1111 1111.1111 1111 / 191.255.255.255
    - 앞의 16비트(옥텟 2개, 맨 앞의 두 자리 십진수 부분)가 네트워크 부분, 나머지 16비트(옥텟 2개, 나머지 2개의 십진수)가 호스트 부분을 나타냄
    - 128 ~ 191으로 시작하는 네트워크
    - 128.0.0.0 ~ 191.255.0.0(네트워크를 나타낼 때 호스트 부분은 모두 0으로 씀) 즉 2^16 - 2 = 65,534개
- 클래스 C
    - ex) 110X XXXX.XXXX XXXX.XXXX XXXX.XXXX XXXX
    - 1100 0000.0000 0000.0000 0000.0000 0000 / 192.0.0.0
    - ex) 1101 1111.1111 1111.1111 1111.1111 1111
    - 앞에 24비트(옥텟 3개, 맨 앞의 3자리 십진수 부분, 나머지 8비트(옥텟 1개, 나머지 1개의 십진수)가 호스트 부분을 나타냄
    - 192.0.0.0 ~ 223.225.225.225 즉 2^8 - 2 = 254개
- 클래스 D : 224 ~ 239(멀티캐스트용 주소)
- 클래스 E : 240 ~ 255(연구용 주소)

## 라우터에서 IP 주소 이해하기

---

- 우리가 보통 쓰는 라우터에 배정해야하는 IP 주소는 2개이다. (네트워크 구성에 따라 달라질 수 있음)
    - 이더넷 인터페이스용 : 사용자가 부여받은 번호 중에 하나를 사용
    - 시리얼(Serial) 인터페이스용 : 사용자가 접속하는 ISP 업체에 따라 다르므로 인터넷 제공업체에 문의하여 사용

## IP 주소의 활용

---

- TCP / IP 통신에서는 라우터의 각 인터페이스에도 IP 주소를 설정하는 것이 좋다
    - 서로 다른 네트워크에 있는 두 장비가 통신하려면 라우터를 거쳐야 한다
- 라우터 인터페이스에 설정하는 IP 주소는 반드시 그 인터페이스가 속한 네트워크 대역의 주소여야 한다
- IP 주소를 배정할 때는 네트워크에 연결될 호스트 수를 먼저 확인해야 한다
- 배정하는 IP 주소 대역이 모든 호스트를 수용할 수 있어야 한다
    - ex) 클래스 C 주소는 최대 254개의 호스트 사용 가능
- 따라서 300개의 호스트가 있는 네트워크에 클래스 C 하나만 배정하면 주소가 부족하다

# 서브넷 마스크

---

- 서브넷(Subnet) : IP 주소를 보다 효율적으로 낭비 없이 쓰기 위해,
적당한 주소를 배정하기 위해
- 서브넷 마스크(Subnet Mask) : 
서브넷을 만들 때 사용하는 마스크
메인이 아닌 어떤 가공을 통한 네트워크를 만들기 위해서 씌우는 마스크
주어진 IP 주소를 네트워크 환경에 맞게 나누어 주기 위해 씌워주는 이진수의 조합
커다란 네트워크를 작게 나우기 위함

## 서브넷 마스크의 기본

---

- 디폴트 서브넷 마스크
    - 네트워크를 작게 나누지 않고, 원래 클래스대로 사용하는 서브넷 마스크
    - 클래스 A의 디폴트 서브넷 마스크 : 255.0.0.0
    - 클래스 B의 디폴트 서브넷 마스크 : 255.255.0.0
    - 클래스 C의 디폴트 서브넷 마스크 : 255.255.255.0
    - 서브넷 마스크는 IP 주소를 가지고 네트워크 부분과 호스트 부분을 알 수 있음
    - IP 주소와 서브넷 마스크 AND 연상을 통해 서브넷 네트워크를 알 수 있음
- 서브네팅
    - 하나의 주소에 서브넷 마스크를 씌워서 작은 네트워크로 만드는 것
    - IP 주소에 디폴트 서브넷 마스크가 아닌 새로운 서브넷 마스크를 씌우면 네트워크 부분이 변경됨
    ex) 클래스 B 주소에 클래스 C의 디폴트 서브넷 마스크를 씌우면 클래스 C처럼 사용한다는 뜻

## 서브넷 마스크의 기본 성질

---

- 서브넷 마스크의 성질
    - 서브네팅을 통해 다른 네트워크가 되면 라우터를 거쳐 통신해야함
    - 서브넷 마스크는 이진수로 썼으르 때 1이 연속적으로 나와야 함
    1의 사이에는 어떠한 경우에도 0이 오면 안 됨
- 네트워크 주소
    - 호스트 부분이 전부 0
- 브로드캐스트 주소
    - 호스트 부분이 전부 1

# 스패닝 트리

---

- 스패닝 트리 알고리즘(SPT, Spanning Tree Protocol)
    - 스위치나 브리지에서 발생하는 루핑을 막아주기 위한 프로토콜
    - 스위치나 브리지 구성에서 출발지부터 목적지까지의 경로가 2개 이상 존재할 때
    1개의 경로만을 남겨두고 나머지는 모두 끊어두었다가,
    사용하던 경로에 문제가 발생하면 끊어둔 경로를 하나씩 살리는 것
- 브리지 ID(Bridge ID)
    - 브리지나 스위치들이 통신할 때 서로를 확인하기 위해 하나씩 가지고 있는 번호
    - Bridge Priority(16비트) + MAC Address(48비트)로 구성
- Path Cost
    - 한 스위치에서 다른 스위치로 가는데 드는 비용
    - 링크의 속도(대역폭)가 빠르면 빠를 수록 Path Cost는 더 작은 값이 됨
    즉, 링크 속도가 빠르면 빨리 도착하므로, Path Cost는 더 적게 들게 됨
    - IEEE는 Path Cost를 정수값으로 지정

## 스패닝 트리를 잘하려면 기억해야할 3가지

---

- 기본 동작 3가지
    1. 네트워크당 하나의 루트 브리지(Root Bridge)를 갖는다
    2. 루트 브리지가 아닌 모든 브리지(Non Root Bridge)는 무조건 하나씩의 루트 포트(Root Port)를 갖는다
    3. 세그먼트(Segment) 당 하나씩의 데지그네이티드 포트(Designated Port)를 갖는다
    
    번외) 스패닝 트리 프로토콜에서 루트 포트와 데지그네이티드 포트가 아닌 나머지 포트는 다 막음
    
- 네트워크당 하나의 루트 브리지(Root Bridge)를 갖는다
    - 네트워크 : 라우터에 의해 나눠지는 브로드캐스트 도메인
    - 하나의 브로드캐스트 도메인에 하나씩 루트 브리지 존대
    - 루트 브리지 : 스패닝 트리 프로토콜을 수행할 때 기준이 되는 브리지(스위치)
- 루트 브리지가 아닌 모든 브리지(Non Root Bridge)는 무조건 하나씩의 루트 포트(Root Port)를 갖는다
    - 루트 포트 : 루트 브리지 쪽에 가장 가까운 포트
- 세그먼트(Segment) 당 하나씩의 데지그네이티드 포트(지정 포트, Designeted Port)를 갖는다
    - 세그먼트 : 브리지 또는 스위치 간에 서로 연결된 링크
    - 브리지나 스위치가 연결되어 있을 때 세그먼트에서 반드시 한 포트는
    Designated Port로 선출되어야 함

# STP 힘 겨루기

---

- 루트 포트와 데지그네이티드 포트 선정 4단계
    1. 누구 더 작은 Root BID를 가졌는가?
    2. 루트 브리지까지의 Path Cost 값은 누가 더 작은가?
    3. 누구의 BID(Sender BID)가 더 낮은가?
    4. 누구의 포트 ID가 더 낮은가?
- BPDU(Bridge Protocol Data Unit)
    - 스패닝 트리 정보를 브리지들 간 주고받기 위해 사용하는 특수한 프레임
    - Root BID, Root Path Cost, Sender BID, Port ID 등이 실려 있음
    - 브리지나 스위치가 부팅을 하면, 브리지(스위치)는 각각의 포트로 BPDU를 매 2초마다 내보내 서로의 스패닝 트리 정보를 주고 받음
    - 어떤 BPDU가 가장 좋은지 결정하기 위해 위의 4단계의 순서 정하기를 사용

# 스위치에서 대장 브리지 뽑기

---

1. 스위치 B, C 부팅
2. 스위치 B, C는 BPDU 주고 받음
BPDU에는 자신의 BID 넣음
3. 스위치는 받은 BPDU에 BID 비교하여 더 낮은 BID를 가진 스위치를 루트 브리지로 선택
32768.2222.2222.2222(스위치 B)가 32768.3333.3333.3333(스위치 C)보다 작으므로 스위치 스위치 B가 루트 브리지
4. 잠시 후 스위치 A 부팅
5. 스위치 A는 BPDU에 자기의 BID를 실어 스위치 B, C에 보냄
6. 스위치 A의 BID가 제일 낮으므로, 스위치 A가 루트 브리지

이때, 스위치 C를 루트 브리지로 만들고 싶다면
Bridge Priority 필드를 수정하여 가장 낮은 BID를 가지게 함

# 데지그네이티드 포트 뽑기

---

- 데지그네이티드 포트
    - 브리지와 브리지로 연결된 세그먼트강 각각 한 개의 데지그네이티드 포트를 뽑아야 함
    - 데지그네이티드 포트는 스패닝 트리에서 어떤 포트를 풀어주고 막을지 결정
    - 세그먼트상에서 Root Path Cost를 서로 비교해
    더 작은 Root Port Cost를 가진 포트가 데지그네이티드 포트로 선정

!image.png

- 데지그네이티드 포트로 선정 과정
    - 세그멘트 1 : 스위치 A의 E0 포트가 데지그네이티드 포트로 선정
    - 세그멘트 2 : 스위치 A의 E1 포트가 데지그네이티드 포트로 선정
    - 세그멘트 3 : 두 포트의 Root Path Cost가 같은 경우 ‘STP에서의 힘 겨루기 4단계’ 거침
- STP에서 힘 겨루기 4단계
    1. 누가 더 작은 Root BID를 가졌는가? → 동점
    두 스위치가 같은 루트 BID(32768.1111.1111.1111) 가짐
    2. 루트 브리지까지의 Path Cost 값은 누가 더 작은가? → 동점
    Root Path Cost가 19로 동일
    3. 누구의 BID(Sender BID)가 더 낮은가? → 스위치 B의 E1 포트 승
    Sender BID : 자기 자신의 BID
    스위치 B의 Sender BID : 32768.2222.2222.2222
    스위치 C의 Sender BID : 32768.3333.3333.3333
    따라서 스위치 B의 Sender BID가 더 낮음
    
    스위치 B의 E1 포트가 데지그네이티드 포트로 선정

# 스패닝 트리 프로토코의 5가지 상태 변화

---

- 스패닝 트리 구성 시 포트의 5가지 상태
    1. Disabled
        - 고장난 포트로 Shut Down 된 상태
    2. Blocking
        - Disabled 되어 있는 포트를 다시 살렸을 때 Blocking 상태
        - 데이터는 전송되지 않고, 오직 BPDU만 주고 받음
    3. Listening
        - Blocking 상태에 이는 포트가 루트 포트나 데지그네이티드 포트로 선정되면 Listening 상태
        - 이때, Non Designated 포트가 되면 Blocking 상태가 됨
    4. Learning
        - Listening 사애에 있던 포트가 포워딩 딜레이 디폴트 시간인 15초 동안 상태를 유지하면 Learning 상태
        - 맥 어드레스를 배워 맥 어드레스 테이블 만듦
    5. Forwarding
        - 포트가 Learning 상태에서 다른 상태가 되지 않고, 다시 포워딩 딜레이 디폴트 시간인 15초 동안 상태를 유지하면 Learning 상태에서 Forwarding 상태
        - Forwarding 상태여야 포트는 데이터 프레임을 주고 받을 수 있음
- 5가지의 상태 변화를 통해 스위치나 브리지는 링크를 마고 열면서 스패닝 트리를 만듦

# 스패니 트리에 변화가 생기던 날

---

- 평화로운 스패닝 트리
    - 루트 브리지는 2초마다 Hello BPDU를 Non Root Bridge에 전송
    - Hello BPDU를 받은 Non Root Bridge는 자신의 데지그네이티드 포트를 통해 다시 전달
    - Non Root Bridge는 지정된 시간 동안 Hello 패킷을 받으면 루트 브리지를 가는 길에 문제 없음을 판단
    만약 문제가 발생됐다고 판단하면 스패닝 트리를 재편성
- 스패닝 트리 재편성 용어 정리
    - Hello Tme
    루트 브리지는 자신에게 연결된 브리지들에게 BPDU를 헬로 타임마다 한 번씩 보냄
    디폴트 헬로 타임은 2초
    - Max Age
    브리지들이 루트 브리지로부터 헬로 패킷을 받지 못하면 멕스 에이지 시간 동안 기다린 후
    스패닝 트리 구조 변경 시작
    디폴트 맥스 에이지는 20초
    - Forwarding Delay
    브리지 포트가 블로킹 상태에서 포워딩 상태로 넘어갈 때까지 걸리는 시간

!image.png

- 스패닝 트리 문제 발생
    - 스위치 A, C 간의 링크에 문제 발생
    - 스위치 C에 2초 후에 헬로패킷이 들어오지 않으면 맥스 에이지 동안 기다림
    맥스 에이지 후 아무런 연락이 없으면 스패닝 트리 변경
    - 스위치 B가 루트 포트(스위치 B의 E0 포트)를 통해 받은 헬로패킷을 다시 데지그네이티드 포트(스위치 B의 E1 포트)를 통해 뿌리고 그 BPDU를 스위치 C의 E1 포트를 통해 받음
    - Blocing 상태인 스위치 C의 E1 포트를 루트 포트로 선정
    스위치 C의 E0 포트는 Blocking 상태가 되미
    - 이때 스위치 C의 E1 포트는 Blocking에서 Listening, Learning을 거쳐 Forwarding 상태로 됨
- 한 링크가 끊어졌을 때, 스패닝 트리 프로토콜을 이용해 다른 경로를 살리는 데 걸리는 시간이 약 50초 정도 소요

# 맥 어드레스는 어디에 저장되어 있을까?

---

- 콜리전 도메인 막는 기능
    - 스위치나 브리지가 출발지에서 들어오는 맥 어드레스를 보고
    그것을 자신의 맥 어드레스 테이블에 저장한 후
     그 주소 테이블에 있는 맥 어드레스를 찾으면 그 쪽 포트로만 보내고. 나머지 포트는 막아줌
- 맥 어드레스를 저장하는 방식
    1. Dynamic 방식
    맥 어드레스 하나를 배우고 나면, 맥 어드레스 테이블에 저장하고 사용
    이때, 이 주소를 사용한지 300초가 지나도록 다시 사용하지 않으면, 
    이 주소는 맥 어드레스 테이블에서 지워짐
    2. Permanent 방식
    천년만년 절대 지워지지 않는 방식
    수동으로 맥 어드레스(스태틱 맥 어드레스)를 저장

# 가상의 랜이란?

---

- 가상의 랜(VLAN, Virtual LAN)
    - 한 대의 스위치를 마치 여러 대의 분리된 스위치처럼 사용 가능하게 함
    따라서 하나의 스위치에 연결된 장비들도 브로드캐스트 도메인이 서로 다를 수 있음
    - 여러 개의 네트워크 정보를 하나의 포트를 통해 전송할 수 있음

즉, 라우터는 스위치로 하나의 링크만을 이용해 3개의 네트워크 정보를 같이 실어 보낼 수 있음

# VLAN에서 꼭 기억해야 할 몇 가지

---

- VLAN에서 기억해야 할 몇 가지
    - VLAN은 스위치에서 지원하는 기능
    VLAN 간의 통신은 오직 라우터를 통해서만 가능
    - 트렁크 포트
    하나의 포트를 통해 서로 다른 여러 개의 VLAN을 전송할 수 있는 포트
- VLAN이 구성된 2대의 스위치
    - 스위치 안에 있는 각각의 스위치를 VLAN이라 함
    따라서 한 대의 스위치가 3개의 VLAN으로 나누어져 있음
    - 2대의 스위치는 패스트 이더넷 포트를 통해 연결
    - VLAN1, VLAN2, VLAN3은 서로 통신 불가
    서로 다른 네트워크이기 때문에 라우터를 통해 가능
    - 같은 VALN끼리는 스위치를 건너서도 통신이 가능

# VLAN에서의 트렁킹과 VTP

---

- 트렁킹
    - 모든 VLAN이 하나의 링크를 통해 다른 스위치나 라우터로 이동시키는 것
- 네이티브 VLAN
    - 모든 VLAN 패킷에 각각 VLAN 정보가 존재하지만, 네이티브 VLAN은 정보가 없음
    - 모든 스위치 네트워크에서 유일하게 한 개의 VLAN만을 네이티브 VLAN으로 세팅 가능
- VTP(VLAN Trunking Protocol) : VLAN 정보를 서로 주고 받아 스위치들이 가지고 있는 VLAN 정보를 항상 일치할 시 켜주는 프로토콜
- VTP 간 주고받는  메시지 형식
    1. Summary Advertisement
    VTP 서버가 자기에게 연결되어 있는 스위치들에게 5분 마다 한 번씩 전달하는 메시지 자신이 관리하는 VTP 도메인의 구성에 대한 Revision 넘버를 보냄
    2. Subset Advertisement
    VLAN의 구성이 변경되었을 때나 VTP 클라이언트로부터 Advertisement Request 메시지를 받을 때 전송
    3. Advertisement Request
    클라이언트가 VTP 서버에 Summay Advertisement와 Subset Advertisement를 요청하는 용도로 사용
- VTP의 3가지 모드
    1. VTP 서버 모드
    2. VTP 클라이언트 모드
    3. VTP 트랜스페어런트 모드
