# 라우터

### 라우터의 정의

라우터(Router)는 **서로 다른 네트워크를 연결하고 데이터를 목적지까지 전달하는 네트워크 장비**이다.

### 라우터의 역할

- 서로 다른 네트워크 연결
- 목적지까지 최적의 경로 선택(라우팅)
- 패킷 전달
- NAT 기능 수행
- 인터넷과 내부 네트워크 연결

### 특징

- OSI 7계층의 **네트워크 계층(Layer 3)** 에서 동작한다.
- IP 주소를 이용하여 목적지를 판단한다.
- 가정에서는 대부분 공유기가 라우터 역할을 함께 수행한다.

# 기본 게이트웨이(Default Gateway)

### 기본 게이트웨이의 정의

기본 게이트웨이는 **내 네트워크에서 다른 네트워크로 나갈 때 가장 먼저 거치는 장치**이다.

### 특징

- 같은 네트워크끼리는 직접 통신한다.
- 다른 네트워크와 통신할 때는 반드시 기본 게이트웨이를 거친다.
- 일반적으로 공유기의 IP 주소가 기본 게이트웨이이다.

### 동작 과정

1. 목적지 IP 확인
2. 같은 네트워크이면 직접 전송
3. 다른 네트워크이면 기본 게이트웨이로 전송
4. 라우터가 목적지까지 전달

# 정적 라우팅(Static Routing)

### 정적 라우팅의 정의

정적 라우팅은 **관리자가 직접 목적지까지의 경로를 설정하는 방식**이다.

### 장점

- 설정이 단순하다.
- CPU와 메모리 사용량이 적다.
- 보안성이 높다.

### 단점

- 네트워크가 커질수록 관리가 어렵다.
- 경로가 변경되어도 자동으로 수정되지 않는다.
- 장애 발생 시 수동으로 변경해야 한다.

### 사용 환경

- 소규모 네트워크
- 실습 환경
- 테스트 환경

# NAT(Network Address Translation)

### NAT의 정의

NAT는 **사설 IP 주소를 공인 IP 주소로 변환하는 기술**이다.

### 사용하는 이유

- 공인 IP 주소 절약
- 내부 IP 주소 보호
- 보안 향상
- 여러 기기가 하나의 공인 IP를 사용할 수 있다.

### NAT의 종류

**Static NAT**

- 사설 IP 1개와 공인 IP 1개를 1:1로 연결

**Dynamic NAT**

- 여러 개의 공인 IP 중 하나를 선택하여 사용

**PAT(NAT Overload)**

- 하나의 공인 IP를 여러 기기가 포트 번호를 이용해 함께 사용
- 가정용 공유기에서 가장 많이 사용되는 방식

# DHCP(Dynamic Host Configuration Protocol)

### DHCP의 정의

DHCP는 **컴퓨터가 네트워크에 연결될 때 필요한 네트워크 정보를 자동으로 설정해 주는 프로토콜**이다.

### DHCP가 제공하는 정보

- IP 주소
- 서브넷 마스크
- 기본 게이트웨이
- DNS 서버 주소

### DHCP 동작 과정(DORA)

**Discover**

- 클라이언트가 DHCP 서버를 찾는다.

**Offer**

- DHCP 서버가 사용할 수 있는 IP 주소를 제안한다.

**Request**

- 클라이언트가 제안받은 IP 주소를 사용하겠다고 요청한다.

**Acknowledge(ACK)**

- DHCP 서버가 사용을 승인한다.

### 장점

- IP 주소를 자동으로 설정한다.
- IP 중복을 방지한다.
- 네트워크 관리가 편리하다.

# DNS(Domain Name System)

### DNS의 정의

DNS는 **도메인 이름을 IP 주소로 변환해 주는 서비스**이다.

### 사용하는 이유

사람은 IP 주소보다 도메인 이름을 기억하기 쉽기 때문에 DNS가 이름을 IP 주소로 변환하여 통신할 수 있도록 한다.

### 동작 과정

1. 사용자가 도메인 입력
2. DNS 서버에 IP 주소 요청
3. DNS 서버가 IP 주소 반환
4. 해당 서버와 통신 시작

### 사용하는 포트

- UDP 53번 : 일반적인 DNS 조회
- TCP 53번 : 영역 전송(Zone Transfer) 및 큰 데이터 전송
